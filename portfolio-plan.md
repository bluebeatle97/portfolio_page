# 포트폴리오 페이지 구성 문서

## 1. 포트폴리오의 핵심 방향

이 포트폴리오는 단순히 프로젝트 결과물을 나열하는 페이지가 아니라,
**기획한 아이디어를 실제 서비스로 구현할 수 있는 사람**이라는 인상을 주는 것을 목표로 한다.

핵심 메시지는 다음과 같다.

> 기획, 연출, 개발, AI 활용, 자동화 도구를 연결해
> 아이디어를 빠르게 검증하고 실제 서비스 형태로 구현하는 개발자

따라서 페이지 전체는 “예쁘게 만든 작업물 소개”보다
**문제 정의 → 기획 의도 → 구현 과정 → 기술적 역할 → 결과물**이 자연스럽게 이어지도록 구성한다.

## 2. 포트폴리오 전체 톤앤매너

### 키워드

* 깔끔함
* 실무적
* 빠른 실행력
* AI 활용 능력
* 기획과 개발의 연결
* 인터랙티브한 느낌
* 과하지 않은 감성

### 피해야 할 느낌

* 지나치게 화려한 애니메이션
* AI가 만든 듯한 과장된 문구
* 추상적인 자기소개
* 프로젝트를 단순 이미지 갤러리처럼 나열하는 구성
* 너무 많은 색상과 장식 요소

### 디자인 방향

전체적으로 흰색 또는 아주 밝은 배경을 기반으로 한다.
포인트 컬러는 블루 계열 또는 블루-퍼플 계열을 사용한다.

시각적으로는 다음 느낌을 목표로 한다.

* 개발자 포트폴리오의 신뢰감
* 기획자의 흐름 설계 능력
* AI 활용자의 실험성과 생산성
* 프로젝트 디렉터의 정리된 사고 방식

## 4-2. About / Positioning Section

### 목적

자기소개를 길게 쓰기보다, 내가 어떤 방식으로 일하는 사람인지 설명한다.

### 문장 예시

저는 아이디어를 기획안에만 남겨두기보다, 실제 화면과 기능으로 구현하는 과정에 관심을 가지고 작업해 왔습니다.
기획, 연출, 개발을 분리된 과정으로 보기보다 하나의 흐름으로 연결하고, AI와 자동화 도구를 활용해 반복 작업을 줄이며 결과물을 빠르게 구체화하는 방식으로 프로젝트를 진행합니다.

### 강조할 포인트

* 기획만 하는 사람이 아님
* 개발만 하는 사람도 아님
* 기획 의도를 이해하고 구현까지 연결할 수 있음
* AI를 단순 생성 도구가 아니라 작업 프로세스 개선 도구로 활용함

### 구성 방식

3개의 카드로 나누어 보여준다.

#### 1. 기획

* 사용자 흐름 설계
* 콘텐츠 구조 설계
* 프로젝트 방향성 정의

#### 2. 구현

* 프론트엔드 화면 구현
* API 데이터 연결
* DB 구조 설계 참여

#### 3. AI 활용

* 이미지·영상 리소스 제작
* LLM API 연동
* Codex 기반 개발 보조
* 테스트 하네스 활용

포트폴리오를 본 사람이 바로 연락하거나 이력서, GitHub, 프로젝트 링크로 이동할 수 있게 한다.

## 포함 요소

* 이름: 김우진
* 이메일
* GitHub
* 이력서 PDF
* 프로젝트 데모 링크
* 블로그 또는 노션 링크가 있다면 추가

## 문장 예시

> 기획과 구현 사이를 연결하는 프로젝트에 관심이 있습니다.
> AI와 자동화 도구를 활용해 아이디어를 빠르게 검증하고 실제 서비스로 만드는 일을 계속해 나가고 싶습니다.

# 11. 페이지 전체 흐름 요약

## 사용자에게 전달되어야 하는 인상

1. 이 사람은 기획을 할 수 있다.
2. 이 사람은 직접 구현도 한다.
3. AI를 단순히 유행처럼 쓰는 것이 아니라 작업 방식에 녹여 쓴다.
4. 프로젝트를 결과물까지 끌고 가는 실행력이 있다.
5. 팀 프로젝트와 개인 프로젝트 모두 경험이 있다.
6. 서비스 구조와 데이터 흐름을 이해하고 있다.

## 가장 중요한 메시지

> 저는 AI를 잘 쓰는 사람이 아니라,
> AI를 활용해 기획, 리소스 제작, 개발, 검증 과정을 연결하고
> 실제로 동작하는 결과물을 만드는 사람입니다.

# 12. Codex 작업 시 주의사항

## 기본 원칙

* 디자인은 깔끔하고 실무적인 느낌으로 유지한다.
* 과도한 애니메이션은 사용하지 않는다.
* 텍스트 가독성을 최우선으로 한다.
* 프로젝트 상세 설명은 카드, 타임라인, 다이어그램 중심으로 구성한다.
* 모바일 반응형을 반드시 고려한다.
* 모든 섹션은 실제 포트폴리오 제출용으로 완성도 있게 구성한다.

## 컴포넌트 구성 예시

* Header
* HeroSection
* AboutSection
* StrengthSection
* FeaturedProjects
* ProjectContactSection
* ColorFitSection
* WorkflowSection
* SkillsSection
* ContactSection
* Footer

## 작업 순서

1. 기본 프로젝트 세팅
2. 전체 레이아웃 구성
3. 디자인 토큰 정의
4. Hero Section 구현
5. About / Strength Section 구현
6. Project Card 구현
7. Project Detail Section 구현
8. Workflow Timeline 구현
9. Skills Section 구현
10. Contact Section 구현
11. 반응형 정리
12. 애니메이션 최소 적용
13. 최종 텍스트 다듬기
14. 배포 준비

# 13. 최종 방향성

이 포트폴리오 페이지는 단순한 자기소개 페이지가 아니라,
**내가 어떤 방식으로 프로젝트를 만들고 문제를 해결하는지 보여주는 페이지**가 되어야 한다.

디자인은 깔끔하게, 문장은 과장 없이, 프로젝트 설명은 구체적으로 구성한다.
특히 AI 활용 역량은 “AI로 만들었다”가 아니라
**AI를 활용해 작업 속도, 리소스 제작, 개발 검증, 서비스 구조 설계까지 연결했다**는 방식으로 표현한다.
