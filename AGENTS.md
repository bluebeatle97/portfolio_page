# AGENTS.md

## Project Role

This repository is a design-focused portfolio landing page project.

The project should prioritize:

- Clear visual hierarchy over excessive animation
- Responsive layouts for mobile, tablet, and desktop
- Section-based growth so new portfolio content can be added over time
- A future admin page for continuously uploading and managing portfolio work

## Agent Guidelines

When working in this repository:

1. Read `README.md`, `DESIGN.md`, and `TASK.md` before making changes.
2. Do not implement broad features without checking the current task scope.
3. Keep changes section-oriented and easy to extend.
4. Prefer simple, readable structure over clever abstractions.
5. Preserve the visual direction described in `DESIGN.md`.
6. Update `TASK.md` when task status, priorities, or implementation notes change.

## Implementation Boundaries

Until the initial documentation phase is complete:

- Do not build the actual landing page.
- Do not scaffold frontend framework files.
- Do not add dependencies.
- Do not create admin page code.

Future implementation should be divided into small, reviewable steps:

- Project setup
- Design system foundations
- Landing page sections
- Portfolio data model
- Admin page
- Upload and content management flow
- Deployment and documentation

## Quality Expectations

Each feature should be checked for:

- Responsive behavior
- Clear content hierarchy
- Accessibility basics
- Maintainable file structure
- Future portfolio item expansion
